API Pedidos v0.4

Documentação completa da API de gerenciamento de pedidos

Versão 0.4

Visão Geral

A API Pedidos v0.4 é uma solução completa para gerenciamento de pedidos, implementada com Node.js, Express, JWT para autenticação e Firebase Realtime Database como banco de dados.

Segurança

Autenticação JWT para endpoints protegidos com tokens válidos por 10 horas.

Armazenamento

Firebase Realtime Database com organização automática de pedidos por data.

Flexibilidade

Suporte a pedidos de entrega e retirada com o campo tipoPedido.

Validação

Validação robusta de dados de entrada com respostas de erro detalhadas.

URL Base

Todas as requisições devem ser feitas para a seguinte URL base:

https://apidelibery.onrender.com

Autenticação

A API utiliza autenticação baseada em tokens JWT (JSON Web Tokens). Para acessar endpoints protegidos, você deve:

  1. Fazer login usando o endpoint /login para obter um token
  2. Incluir o token no cabeçalho Authorization de todas as requisições subsequentes

Formato do cabeçalho:

Authorization: Bearer SEU_TOKEN_JWT_AQUI

Os tokens têm validade de 10 horas.

Criar Pedido

Esquema do Pedido

Campo Tipo Obrigatório Descrição
cliente string Sim Nome do cliente
telefone string Não Telefone do cliente
pagamento string Não Forma de pagamento (Pix, Cartão, Dinheiro, Outros)
taxa number Não Taxa de entrega
valor_total number Não Valor total do pedido
tipoPedido NOVO string Não Tipo do pedido: "Entrega" ou "Retirada" (padrão: "Entrega")
endereco object Sim Endereço de entrega (obrigatório mesmo para retirada)
endereco.rua string Não Nome da rua
endereco.numero string Não Número do endereço
endereco.bairro string Não Bairro
endereco.referencia string Não Ponto de referência
itens object Não Itens do pedido
motoboy object Não Informações do motoboy

Pedidos por Data

GET /pedidos/:data Protegido

Retorna todos os pedidos de uma data específica. O parâmetro :data deve estar no formato DDMMAAAA.

Requisição
Resposta

Cabeçalhos

Authorization: Bearer SEU_TOKEN_JWT_AQUI

Parâmetros de URL

Parâmetro Tipo Obrigatório Descrição
data string Sim Data no formato DDMMAAAA (ex: 26112025)

Exemplo de URL

GET https://apidelibery.onrender.com/pedidos/26112025

Resposta de Sucesso (200 OK)

{
  "-Nr98h12ad9": {
    "cliente": "João Silva",
    "endereco": { ... },
    "tipoPedido": "Entrega",
    "id": 0,
    "status": "pendente",
    "valor_total": 42.90
  },
  "-Nr98h34bc0": {
    "cliente": "Maria Santos",
    "endereco": { ... },
    "tipoPedido": "Retirada",
    "status": "entregue",
    "valor_total": 35.50
  }
}

Resposta para Data sem Pedidos (200 OK)

{}

Exemplos de Uso

Exemplo 1: Autenticação e Criação de Pedido de Entrega

// 1. Fazer login para obter token
const loginResponse = await fetch('https://apidelibery.onrender.com/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    usuario: 'admin',
    senha: '123'
  })
});

const { token } = await loginResponse.json();

// 2. Criar um pedido de entrega usando o token
const pedidoResponse = await fetch('https://apidelibery.onrender.com/pedido', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    cliente: 'João Silva',
    telefone: '11999999999',
    pagamento: 'Pix',
    tipoPedido: 'Entrega',
    id: 0,
    taxa: 5,
    valor_total: 42.90,
    endereco: {
      rua: 'Rua das Flores',
      numero: '123',
      bairro: 'Centro',
      referencia: 'Próximo à praça'
    },
    itens: {
      item1: {
        nome: 'Pizza Margherita',
        qtd: 1,
        preco: 42.90
      }
    }
  })
});

const resultado = await pedidoResponse.json();
console.log(resultado);

Exemplo 2: Criar Pedido de Retirada

// Pedido de retirada (tipoPedido: "Retirada")
const pedidoRetirada = {

  },

      nome: 'Hambúrguer Artesanal',

// Enviar requisição

  method: 'POST',

    'Content-Type': 'application/json',

  body: JSON.stringify(pedidoRetirada)
});

Exemplos cURL

Exemplo 3: Listar Pedidos com cURL

# Obter token
TOKEN=$(curl -s -X POST https://apidelibery.onrender.com/login \
  -H "Content-Type: application/json" \
  -d '{"usuario":"admin","senha":"123"}' | jq -r '.token')

# Listar pedidos do dia
curl -H "Authorization: Bearer $TOKEN" \
  https://apidelibery.onrender.com/pedidos

# Listar pedidos de uma data específica
curl -H "Authorization: Bearer $TOKEN" \
  https://apidelibery.onrender.com/pedidos/26112025

Códigos de Status

Código Descrição
200 OK - Requisição bem-sucedida
201 Created - Recurso criado com sucesso
400 Bad Request - Dados da requisição inválidos
401 Unauthorized - Token inválido ou não fornecido
500 Internal Server Error - Erro no servidor
